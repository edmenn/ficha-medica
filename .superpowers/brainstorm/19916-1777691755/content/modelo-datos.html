Decisión de diseño clave

3 capas de datos por registro:

① ai_raw_response — respuesta cruda de OpenRouter (nunca se toca)

② extracted_data — JSON estructurado parseado de la IA

③ final_data — datos corregidos por el usuario (lo que se usa para reportes)

Esto garantiza trazabilidad completa: siempre se puede ver qué dijo la IA vs qué corrigió el médico.
